# Pupbox 项目风险评审

更新时间：2026-07-12。目标形态是一只放在家中、由三岁孩子直接使用、无屏幕的联网毛绒小狗。本评审区分“当前家长陪同的网页原型”和“最终儿童硬件”，后者需要更严格的可靠性、隐私和设备安全要求。

## 当前线上测量

日本 VPS 为 2C、约 1GB 内存。检查时系统负载约 0.07，Pupbox RSS 约 15MB，可用内存约 524MB，并有 1GB Swap；当前规格足够运行 Go 服务、静态文件、短期会话和 JSONL 日志。

最近 47 个有效语音事件的近似分位值：

| 指标 | P50 | P90 |
| --- | ---: | ---: |
| 松开录音到可播放语音 | 1.77s | 5.02s |
| STT | 1.10s | 2.27s |
| Qwen/回复 | 0.69s | 1.38s |
| TTS 文件可播放 | 1.21s | 4.47s |
| 完整一轮，含录音和播放 | 8.85s | 15.18s |

结论：CPU 和内存不是当前延迟主因。主要问题是 STT、TTS 长尾，以及“中国手机 -> 日本 VPS -> 阿里云北京接入点”的跨地域链路。日本 VPS 到 `dashscope.aliyuncs.com` 的一次无推理 HTTPS 探测约需 0.62 秒完成 TLS、0.74 秒收到响应，说明地域链路本身已经占用可感知时间。

## 模型与成本

当前组合是 `qwen3-asr-flash`、`qwen-plus-character` 和 `cosyvoice-v3-flash`。阿里官方将 Qwen Character 定位为适合角色扮演、虚拟社交、游戏 NPC、硬件和玩具的模型，因此在真实对话没有明显退化前，不建议仅因为“海外模型可能更强”就整体替换。

按每轮 4 秒录音、500 个输入 Token、60 个输出 Token、50 个 TTS 汉字粗略估算 1000 轮：中国内地 ASR 约 0.88 元、Qwen Character 约 0.52 元、CosyVoice 约 5 元，总计约 6.4 元，不含 VPS 和流量；固定回复命中 TTS 缓存后还会更低。TTS 是主要变量，文本模型费用很小。

新加坡国际部署按同样假设约 8.3 元/1000 轮。价格仍然可接受，但不同地域的 API Key、模型列表和接入域名不能混用。国际部署不必然降低中国家庭到服务端的链路长尾，必须用真实家庭网络 A/B 测量。

OpenAI 可以作为“仅聊天模型”的对照组，而不是立即替换整条语音链路。GPT-4o mini 的文本单价为每百万输入/输出 Token 0.15/0.60 美元，同样假设约 0.11 美元/1000 轮；成本不高，但还要考虑单独充值、跨境延迟、儿童中文语境表现和供应商故障。下一步应使用同一批匿名化对话做盲评，比较“是否接住、是否像三岁陪玩、是否过长”，而不是凭模型品牌切换。

## 是否需要家庭 VPN

不需要，也不应该把 VPN 作为儿童玩具的正常依赖。最终 ESP32 只需连接家中 2.4GHz Wi-Fi，并通过标准 HTTPS 443 访问 `pupbox.983457.xyz`。V2Ray 是 VPS 上的另一项服务，与 Pupbox 客户端无关。

海外域名在中国家庭网络下可能出现抖动或个别运营商路由问题，但正确做法是测量并准备服务端迁移、国内/香港接入或备用域名，而不是要求玩具加入 VPN。VPN 会增加配置、掉线、证书、合规和家长维护成本。

## 最终硬件前的高优先级风险

1. **设备鉴权和费用保护**：当前是一个共享 Bearer Token，没有设备级撤销、速率限制和每日费用上限。最终硬件需要每台设备独立凭证、服务端限流和预算告警。
2. **儿童隐私**：诊断录音会短期保存在 VPS。需要默认最短保留、家长明确开启、可一键清除，并确认云厂商地域和语音数据处理范围。不要长期保存原始儿童语音。
3. **单点故障**：VPS、域名、阿里 API 和家庭网络任何一处失败都会让玩具沉默。硬件必须内置“网络不好，豆豆休息一下”等离线语音，并能恢复重试。
4. **会话易失**：当前上下文只在内存保存最近 10 轮、30 分钟，服务重启即丢失。短期陪玩足够；最终如需记住偏好，应只保存家长允许的非敏感偏好，不保存完整儿童画像。
5. **中断与回声**：虽然本轮不改网页按钮，最终硬件仍必须有物理停止方式。免按键阶段还需要 AEC/回声抑制，否则扬声器声音会触发自己的麦克风。
6. **同机运行 VPN**：当前负载足够，但 Pupbox 与 V2Ray 共用 VPS 会共享网络、磁盘和故障域。进入家庭长期使用前，至少增加服务监控、资源限制和独立备份；用户量增加后再拆机。
7. **模型安全不是绝对保证**：本地危险词规则只能覆盖已知表达，LLM 也可能漂移。需要持续加入真实儿童说法的回归集，并保留家长陪同测试和内容类别开关。
8. **设备运维**：ESP32 需要安全的 Wi-Fi 配网、证书时间校验、签名 OTA、物理静音/关机、故障指示和恢复出厂设置。固件中不能硬编码仓库访问令牌或共享 API Key。

## 推荐决策顺序

1. 先用新增的评价和 P50/P90 连续收集一到两个周末数据，确认 P90 慢轮次来自 STT、Qwen、TTS 还是家庭到 VPS 网络。
2. 保持阿里语音链路不变，对聊天模型做离线 A/B；只有盲评稳定胜出才增加第二在线 Provider。
3. 购买 ESP32-S3 音频板后继续访问现有 HTTPS 服务，不加入 VPN。
4. 桌面硬件验证前补设备级鉴权、限流、离线错误语音和诊断中的设备版本字段。
5. 毛绒外壳测试前补物理停止/静音、录音指示、AEC 和儿童录音清理能力。
